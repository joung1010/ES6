## Set 데이터의 집합체
  인덱스 , 순서가 없다   
  그냥 하나의 집합체 하지만 **중복 불가**   
  ### 선언
<pre>
<code>
const set = new Set([1,2,3]);
console.log(set); //Set(3) { 1, 2, 3 }
</code>
</pre>
### 메서드
<pre>
<code>
const set = new Set([1,2,3]);

//사이즈 확인(size)
console.log(set.size);

// 해당 값이 존재하는지 확인(has)
console.log(set.has(2));
console.log(set.has(6));

// 순회
set.forEach((item) => console.log(item));
for (const value of set.values()) {
    console.log(value);
}

//추가 (add) (중복 값은 무시됨)
set.add(6);
console.log(set); //Set(4) { 1, 2, 3, 6 }
set.add(6);
console.log(set); //Set(4) { 1, 2, 3, 6 }

//삭제 (delete)
set.delete(6);
console.log(set);

// 전부 삭제(clear)
set.clear();
console.log(set);
</code>
</pre>
오브젝트 타입 set
<pre>
<code>
const obj1 = {name: 'apple', price: 4};
const obj2 = {name: 'banana', price: 5};
const objs = new Set([obj1, obj2]);
console.log(objs); //Set(2) { { name: 'apple', price: 4 }, { name: 'banana', price: 5 } }

obj1.price = 10;
console.log(obj1);
console.log(objs);
</code>
</pre>
여기서 obj1의 price를 변경하면 objs 안에 있는 값은 변결될까??   
정답은 Yes   
shallow-copy 가 발생하기 때문이다.
